## 1. 接口规范

### 1.1 基础规范

| 规范项 | 说明 |
|--------|------|
| 接口风格 | RESTful |
| 通信协议 | HTTPS |
| 数据格式 | JSON（UTF-8编码） |
| 接口前缀 | `/api/v1` |
| 鉴权方式 | JWT Bearer Token |
| 时间格式 | `yyyy-MM-dd HH:mm:ss` |
| 分页参数 | `pageNum`（页码，从1开始）、`pageSize`（每页条数，默认10） |

### 1.2 请求方法约定

| HTTP方法 | 用途 | 示例 |
|---------|------|------|
| GET | 查询资源 | `GET /api/v1/identification/tasks` |
| POST | 创建资源 | `POST /api/v1/identification/tasks` |
| PUT | 更新资源（全量） | `PUT /api/v1/identification/tasks/{id}` |
| PATCH | 更新资源（部分） | `PATCH /api/v1/identification/tasks/{id}/status` |
| DELETE | 删除资源 | `DELETE /api/v1/identification/tasks/{id}` |

### 1.3 统一响应格式

**成功响应：**

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "id": 1234567890,
    "name": "示例数据"
  },
  "timestamp": 1719216000000
}
```

**分页响应：**

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "list": [
      { "id": 1, "name": "记录1" },
      { "id": 2, "name": "记录2" }
    ],
    "total": 100,
    "pageNum": 1,
    "pageSize": 10,
    "pages": 10
  },
  "timestamp": 1719216000000
}
```

**错误响应：**

```json
{
  "code": 40001,
  "message": "参数校验失败：ISBN不能为空",
  "data": null,
  "timestamp": 1719216000000,
  "errors": [
    {
      "field": "isbn",
      "message": "ISBN不能为空"
    }
  ]
}
```

### 1.4 请求头规范

| 请求头 | 是否必填 | 说明 |
|--------|---------|------|
| Authorization | 是（除登录接口） | `Bearer {JWT_TOKEN}` |
| Content-Type | 是 | `application/json;charset=UTF-8` |
| Accept-Language | 否 | 语言，默认`zh-CN` |
| X-Request-Id | 否 | 请求追踪ID，未传则自动生成 |
| X-Tenant-Id | 否 | 租户ID（多租户场景） |

### 1.5 接口命名规范

- 资源路径采用复数形式：`/tasks`、`/merchants`
- 路径层级不超过3层：`/api/v1/identification/tasks/{id}/materials`
- 操作动词放在路径末尾：`/tasks/{id}/assign`、`/tasks/{id}/submit`

**接口描述：** 用户登录，获取JWT Token

## 附录：接口版本管理

### 版本策略

1. **URL版本**：通过URL路径区分版本，如`/api/v1/`、`/api/v2/`
2. **向后兼容**：新版本必须向后兼容旧版本至少6个月
3. **废弃通知**：废弃接口在响应头添加`X-Deprecated: true`和`Sunset`头
4. **变更日志**：每次接口变更需更新变更日志
